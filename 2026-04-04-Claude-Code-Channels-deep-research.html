一、Claude Code Channels 官方功能介绍与核心定位

1.1 什么是 Claude Code Channels

Claude Code Channels 是 Anthropic 于 2026年3月 正式发布的一项研究预览（Research Preview） 功能，属于 Claude Code 产品生态中的跨平台消息接入层。其核心功能是：将外部平台（Telegram、Discord、iMessage 等）的消息和事件推送到一个运行中的本地 Claude Code 会话，让 Claude 能够实时响应外部触发源。

Push events into a running session with channels — Use channels to push messages, alerts, and webhooks into your Claude Code session from an MCP server. Forward CI results, chat messages, and monitoring events so Claude can react while you're away.

1.2 核心功能

Channels 本质上是一个 MCP（Model Context Protocol）服务器——它不是 Claude Code 的替代品，而是一个事件注入管道：

功能说明
Telegram 集成用户通过 BotFather 创建 Bot，在 Telegram 中向 Bot 发送消息，消息被注入 Claude Code 会话，Claude 的回复通过 Bot 传回 Telegram
Discord 集成创建 Discord Bot，开通 Message Content Intent，邀请 Bot 加入服务器，DM Bot 即可触发 Claude Code 会话
iMessage 集成读取 macOS 本地 Messages 数据库（~/Library/Messages/chat.db），通过 AppleScript 回复，支持自发自收
双向通信Claude 读取外部事件并回复，回复显示在对方平台而非终端
发送者白名单可配置仅允许特定账户发送消息，防止未授权访问
自定义 Channel提供 Channels API，允许开发者构建自己的 Channel（不只是聊天平台，可接入 CI 系统、监控系统等）

技术要求：

  • Claude Code v2.1.80 或更高版本
  • claude.ai OAuth 登录（不支持 Console 或 API Key 认证）
  • Team/Enterprise 组织需管理员在 claude.ai/admin-settings/claude-code 显式启用
  • Bun 运行时（用于运行 Channel 插件）

启动方式：

claude --channels plugin:telegram@claude-plugins-official

三、技术原理深度解析

3.1 Claude Code Channels 工作原理

[外部平台] → [Channel Plugin (Bun)] → [Claude Code 本地会话] → [回复路由] → [外部平台]
(Telegram/   (MCP Server,        (会话运行在        (工具调用，      (Bot/Discord Bot
 Discord/    轮询或长连接)       用户本地机器)       回复发回 Channel)  回复用户)
 iMessage)

关键点：

  1. Channel 本质是一个 MCP Server，而不是传统意义上的 Bot 框架
  2. Claude Code 运行在用户本地，Channel Plugin（Bun 进程）通过轮询（polling）或 WebSocket 接收外部消息
  3. 消息通过 Anthropic API 的流式连接路由到本地 Claude Code 会话
  4. Claude 的回复通过同一个 Channel 传回外部平台
  5. 会话必须在终端保持运行——关闭终端 = Channel 断开

启动方式决定架构：

  • claude --channels plugin:telegram@... — 交互式模式，同时支持本地和远程
  • claude remote-control — Server 模式，纯远程，可后台运行

3.2 Remote Control vs Dispatch vs Channels 的技术差异

这三个功能都实现了"跨设备控制"，但原理不同：

维度Remote ControlDispatchChannels
控制目标Claude Code 桌面会话Cowork 桌面会话Claude Code 桌面会话
交互界面claude.ai/code 网页 / Claude App（iOS/Android）Claude App（iOS/Android）Telegram/Discord/iMessage
技术架构WebSocket 流式连接，本地会话轮询 Anthropic APICowork 持久会话 + QR 码配对MCP Server（Plugin），Bun 进程轮询外部 API
会话保持本地进程必须运行，断线自动重连本地 Desktop App 必须运行同样，本地 Claude Code 必须运行
安全模型OAuth 短效凭证，超时会话退出沙箱执行，用户审批白名单 + 配对码

共同限制： 三者都将桌面机器作为计算引擎，手机只是遥控器。关闭笔记本、睡眠、断网 = 功能中断。

3.3 Anthropic 的"平台整合"战略

Claude Code Channels 的发布是 Anthropic 在 2026 年初对 OpenClaw 展开的"平台挤压（Platform Squeeze）"战略的最后一步。该战略分为四个阶段：

  1. 品牌隔离（2026年1月）：商标侵权警告，强制将"Clawdbot"更名为"Moltbot"再改为"OpenClaw"
  2. 经济断血（2026年1-2月）：封禁第三方 OAuth 使用低价订阅 token，迫使用户切换到高价 API 计费
  3. 功能克隆（2026年2-3月）：四周内上线 Cron（Scheduled Tasks）、Memory（Auto Memory）、Plugins（Plugin System）、Remote Control
  4. Channels 收官（2026年3月）：复制 OpenClaw 最后一项独特能力——通过聊天平台远程控制本地 Agent

五、用户和开发者社区反馈

5.1 2026年3月限额危机（严重影响用户体验）

2026年3月23日起，大量 Claude Code 用户反映：

  • 10-15分钟内耗尽会话限额（原本设计为数小时）
  • Max 计划用户（$100-200/月）在单日下午耗尽整周限额
  • GitHub 和 Reddit 出现大量报告帖
  • Anthropic 承认存在异常，曾在3月26日短暂限制高峰时段配额
  • 3月31日 The Register 报道 Anthropic 承认限额消耗"远超预期"
⚠️ 关键风险："质量再好，无法使用也无意义。" — 有用户表示 Claude Code 质量优异，但 API 限额异常严重影响正常使用。

5.2 Channels 的早期评价

正面反馈：

  • 开发者对"用手机发 Telegram 消息触发 Claude Code 执行"的能力表示认可
  • iMessage 集成受到 macOS 用户好评（无需 Bot Token，直接读取本地数据库）
  • 自定义 MCP Channel 的扩展性被开发者肯定

负面反馈：

  • 研究预览状态意味着不稳定和无 SLA 保证
  • 目前支持的平台太少（相比 OpenClaw 的 30+）
  • 与 OpenClaw 的功能重叠被社区描述为"模仿而非创新"
  • 安装依赖 Bun 增加了门槛（OpenClaw 只需要 Node.js）

5.3 Claude Code vs OpenClaw 社区对比评价

Claude Code 被高频称赞的点：

  • 深度代码上下文理解（200K-1M token 窗口）
  • 无需基础设施配置，安装即用
  • IDE 插件（VS Code/JetBrains）的无缝集成
  • PR 创建、测试执行、git 工作流的完整内置
  • Anthropic 安全团队托管的沙箱模型

Claude Code 被高频抱怨的点：

  • Session 记忆重置需要反复描述项目背景
  • 复杂任务（超过单一明确指令）成功率低
  • 价格（$20-200/月）对于非编程用途显得昂贵
  • API 限额问题影响正常使用

OpenClaw 被高频称赞的点：

  • 多平台消息接入（WhatsApp/Telegram/Discord/Slack）
  • 持久记忆，跨会话上下文不丢失
  • 完全免费（使用本地模型）
  • 任务执行时用户可以离开（"fire and forget"）
  • Skills 生态的丰富性（5700+）

OpenClaw 被高频抱怨的点：

  • CVE-2026-25253 安全事件后社区信任度下降
  • ClawHub 恶意 Skills 问题（824+ 个恶意 Skills，20%）
  • 初始配置耗时（"setup weekend is real"）
  • 创始人离开项目的影响不确定
  • 代码执行能力不如 Claude Code 精细

5.4 "平台挤压"事件的社区反应

Epsilla 等分析文章将 Anthropic 的行为定性为"Platform Squeeze"（平台挤压）：

  • 社区对 Anthropic 的快速跟进持复杂态度
  • 部分开发者认为"这是正常的市场竞争，Anthropic 有权完善自己的产品"
  • 另一部分认为"四周内完整克隆一个开源项目的核心功能，不够体面"
  • OpenClaw 创始人 Peter Steinberger 本人未公开评论（已加入 OpenAI）
  • OpenClaw 社区仍在活跃（257K+ GitHub Stars，1200+ 贡献者，11440+ commits），项目生命力依然强劲

六、综合结论

Claude Code Channels 是 Anthropic 在 2026 年初对 OpenClaw 展开的系统性产品反击的一部分。它在技术上是一个基于 MCP 协议的 Channel 插件系统，通过 Bun 运行时将 Telegram/Discord/iMessage 的消息注入本地 Claude Code 会话。它解决了 Claude Code 用户长期缺乏的"移动端触发"能力，但作为研究预览，功能深度和平台覆盖都明显不足。

与 OpenClaw 相比，两者代表了 AI Agent 的两条根本不同的路径：

  • Claude Code = 专业化深度（编程为主，平台受控，安全托管）
  • OpenClaw = 通用化广度（多平台、多模型、自托管）

OpenClaw 在消息平台接入、持久记忆、自托管成本和模型灵活性上仍有显著优势；Claude Code 在代码理解深度、安全性、用户体验和模型能力上领先。两者不是直接竞争对手——至少在 OpenClaw 转向独立基金会后，市场正在重新定义边界。

⚠️ 关键风险提示： Claude Code 当前正经历严重的 API 限额消耗异常（2026年3月23日起），已影响大量付费用户的正常使用，这是选择该平台时需要关注的重要变量。
